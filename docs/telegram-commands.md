# 텔레그램 명령어 레퍼런스

## 명령어 목록

| 명령어 | 설명 | 위험도 |
|--------|------|--------|
| `/start` | 봇 연결 확인 및 도움말 | 안전 |
| `/help` | 명령어 도움말 | 안전 |
| `/status` | 현재 시스템 상태 조회 | 안전 |
| `/settings` | 현재 설정값 조회 | 안전 |
| `/logs [N]` | 최근 N개 로그 조회 (기본 10) | 안전 |
| `/trades [N]` | 최근 N개 매매 기록 조회 (기본 10) | 안전 |
| `/pyramid_status` | 피라미딩 포지션 상태 조회 | 안전 |
| `/pyramid_set [마켓] [진입가] [횟수]` | 피라미딩 상태 수동 설정 | 중간 |
| `/sync` | 업비트 잔고 기준 피라미딩 상태 동기화 | 중간 |
| `/strategy [이름]` | 전략 변경 (핫 리로드) | 중간 |
| `/backtest [전략] [일수]` | 백테스트 실행 | 안전 |
| `/block [마켓] [사유]` | 특정 마켓 매수 금지 등록 | 중간 |
| `/unblock [마켓]` | 매수 금지 해제 | 중간 |
| `/blocked` | 매수 금지 목록 조회 | 안전 |
| `/sell [마켓]` | 보유 종목 선택 매도 (생략 시 목록 표시) | 높음 |
| `/halt` | 매매 중단 (킬스위치 발동) | 높음 |
| `/resume [마켓]` | 킬스위치 해제 (마켓 지정 시 해당 마켓만 해제) | 높음 |
| `/panic_sell` | 긴급 전량 시장가 매도 | 매우 높음 |

---

## 상세 설명

### `/status`
현재 시스템 상태 요약을 즉시 보고합니다.

```
📊 시스템 상태

매매 상태: 🟢 운영 중
현재 전략: pyramid_breakout
총 자산: 1,234,567원
운영 모드: 모의

차단된 마켓: 없음
```

오른쪽 아래 **🔄 새로고침** 버튼으로 즉시 갱신할 수 있습니다.

---

### `/settings`
현재 적용 중인 설정값 전체를 표시합니다. `.env` 파일 수정 없이 현재 상태를 빠르게 확인할 때 사용합니다.

---

### `/trades [N]`
최근 N건(기본 10)의 매매 기록과 30일 성과 요약(총 거래 수, 승률, 누적 손익)을 함께 표시합니다.

---

### `/pyramid_status`
현재 추적 중인 피라미딩 포지션의 진입가·추가매수 횟수·다음 추가매수 기준가를 표시합니다.

---

### `/pyramid_set [마켓] [진입가] [횟수]`
피라미딩 포지션 상태를 수동으로 등록합니다. `/sync`로 자동 동기화가 안 되는 경우 사용합니다.

```
/pyramid_set KRW-BTC 95000000 2
→ ✅ 피라미딩 상태 설정 완료
   마켓: KRW-BTC
   진입가: 95,000,000원
   추가매수 완료: 2회
   다음 추가매수 기준: 104,500,000원 (10% 상승)
```

---

### `/sync`
업비트 실제 잔고를 조회하여 피라미딩 상태를 자동으로 동기화합니다.
잔고가 없는 마켓은 제거하고, 잔고가 있는 마켓은 평균단가 기준으로 추가합니다.

---

### `/block [마켓] [사유]`
특정 마켓을 매수 금지 목록에 추가합니다. 이후 해당 마켓의 신규 매수·추가매수가 차단됩니다.
사유는 선택사항이며 `/blocked`에서 함께 표시됩니다.

```
/block KRW-SHIB 고점 판단
→ 🚫 KRW-SHIB 매수 금지 등록 완료
   사유: 고점 판단
```

`.env`의 `EXCLUDED_MARKETS`로 시작부터 특정 마켓을 정적으로 제외할 수도 있습니다.

---

### `/unblock [마켓]`
매수 금지를 해제합니다. DB에서도 삭제되어 재시작 후에도 유지됩니다.

```
/unblock KRW-SHIB
→ ✅ KRW-SHIB 매수 금지 해제 완료
```

---

### `/blocked`
현재 등록된 매수 금지 마켓 목록과 사유를 표시합니다.

---

### `/sell [마켓]`
보유 중인 종목을 시장가로 즉시 매도합니다.

**마켓을 지정하지 않으면** 현재 보유 종목 목록이 인라인 버튼으로 표시됩니다:

```
📊 매도할 종목을 선택하세요
[KRW-BTC] [KRW-ETH]
[❌ 취소]
```

**마켓을 직접 지정하면** 확인 버튼이 바로 표시됩니다:

```
/sell KRW-BTC
→ 🔴 KRW-BTC 매도 확인
   시장가로 전량 매도하시겠습니까?
   [✅ 매도 실행] [❌ 취소]
```

---

### `/strategy [이름]`
서버 재시작 없이 매매 전략을 변경합니다.

```
/strategy pyramid_breakout
→ ✅ 전략이 pyramid_breakout으로 변경되었습니다.
   서버 재시작 없이 즉시 적용됩니다.
```

사용 가능한 전략 목록 확인:
```
/strategy
→ 사용 가능한 전략: momentum, mean_reversion, pyramid_breakout, ...
```

---

### `/halt`
모든 신규 주문을 즉시 차단합니다. 이미 진입된 포지션은 유지됩니다.
텔레그램에서 확인 버튼을 눌러야 실제 발동됩니다.

```
⚠️ 매매를 중단하시겠습니까?
[✅ 확인 - 매매 중단] [❌ 취소]
```

---

### `/resume [마켓]`
킬스위치를 해제하고 매매를 재개합니다.

**마켓 없이 사용** → 매크로·수동 킬스위치 전체 해제:
```
/resume
→ ✅ 킬 스위치 전체 해제 완료. 매매를 재개합니다.
```

**마켓 지정** → 해당 마켓의 마이크로 킬스위치만 해제:
```
/resume KRW-BTC
→ ✅ KRW-BTC 킬스위치 해제 완료.
```

차단 상태가 아닌 마켓을 지정하면 안내 메시지가 반환됩니다.

---

### `/panic_sell`
⚠️ **가장 위험한 명령어** - 모든 보유 코인을 즉시 시장가로 매도합니다.

시장의 비정상적 폭락이 예상될 때만 사용하세요.
확인 버튼을 눌러야 실제 실행됩니다.

```
🚨 긴급 전량 매도
⚠️ 모든 보유 코인을 즉시 시장가로 매도합니다.
이 작업은 되돌릴 수 없습니다!

[🚨 전량 매도 실행] [❌ 취소]
```

## 자동 알림

다음 이벤트에서 자동으로 알림이 전송됩니다:

- **킬스위치 발동**: 포트폴리오 낙폭이 임계치 초과 시
- **손절 발동**: 개별 코인이 손절 기준선 하락 시
- **수동 halt 발동**: 텔레그램 명령으로 중단 시
- **거래지원 종료 예정 감지**: 보유/미보유 구분하여 알림 (최초 감지 시 1회만)
  - 보유 중인 경우: 자동으로 시장가 매도하고 매수 금지 등록
  - 미보유인 경우: 알림 후 신규 매수 차단
- **원화 잔고 부족**: `MIN_KRW_ALERT` 이하 시 5분마다 경고 (중복 억제)
- **에러 발생**: 시스템 치명적 오류 시
